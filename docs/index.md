# Supermetal Provider

The Supermetal provider manages [CDC connectors](https://docs.supermetal.io/docs/main/) on
[Supermetal](https://supermetal.io) agents. Each connector replicates data from a source
database to a destination target.

## Supported sources

[PostgreSQL](https://docs.supermetal.io/docs/main/sources/pg),
[MySQL](https://docs.supermetal.io/docs/main/sources/mysql),
[SQL Server](https://docs.supermetal.io/docs/main/sources/sqlserver),
[Oracle](https://docs.supermetal.io/docs/main/sources/oracle),
[MongoDB](https://docs.supermetal.io/docs/main/sources/mongo),
[File (S3, GCS, Azure Blob)](https://docs.supermetal.io/docs/main/sources/file)

## Supported targets

[Snowflake](https://docs.supermetal.io/docs/main/targets/snowflake),
[BigQuery](https://docs.supermetal.io/docs/main/targets/bigquery),
[Databricks](https://docs.supermetal.io/docs/main/targets/databricks),
[ClickHouse](https://docs.supermetal.io/docs/main/targets/clickhouse),
[Apache Iceberg](https://docs.supermetal.io/docs/main/targets/iceberg),
[Apache Kafka](https://docs.supermetal.io/docs/main/targets/kafka),
[DuckDB](https://docs.supermetal.io/docs/main/targets/duckdb),
[MotherDuck](https://docs.supermetal.io/docs/main/targets/motherduck),
[PostgreSQL](https://docs.supermetal.io/docs/main/targets/postgres),
[Apache Doris](https://docs.supermetal.io/docs/main/targets/doris),
[Webhook](https://docs.supermetal.io/docs/main/targets/webhook)

## Prerequisites

A running Supermetal agent. See [Deployment](https://docs.supermetal.io/docs/main/concepts/deployment)
for installation options (local binary, Kubernetes Helm chart, or Terraform module for AWS).

## Example Usage

```terraform
terraform {
  required_providers {
    supermetal = {
      source = "supermetal-inc/supermetal"
    }
  }
}

variable "supermetal_username" {
  type      = string
  sensitive = true
}

variable "supermetal_password" {
  type      = string
  sensitive = true
}

provider "supermetal" {
  endpoint = "https://sm.internal:3000"
  username = var.supermetal_username
  password = var.supermetal_password
}

# Multiple agents via provider aliases
provider "supermetal" {
  alias    = "analytics"
  endpoint = "https://sm-analytics.internal:3000"
  username = var.supermetal_username
  password = var.supermetal_password
}
```

## Authentication

The provider supports HTTP basic authentication. Credentials can be set
in the provider block or via environment variables.

| Attribute | Environment Variable |
|-----------|---------------------|
| `endpoint` | `SUPERMETAL_ENDPOINT` |
| `username` | `SUPERMETAL_USERNAME` |
| `password` | `SUPERMETAL_PASSWORD` |
| `ca_cert` | `SUPERMETAL_CA_CERT` |
| `skip_validation` | `SUPERMETAL_SKIP_VALIDATION` |

## Schema

### Optional

- `ca_cert` (String) PEM-encoded CA certificate for TLS verification. Can also be set via `SUPERMETAL_CA_CERT` environment variable.
- `endpoint` (String) Supermetal agent API endpoint (e.g. `https://sm.internal:3000`). Can also be set via `SUPERMETAL_ENDPOINT` environment variable.
- `insecure` (Boolean) Skip TLS certificate verification. Not recommended for production.
- `password` (String, Sensitive) Password for basic authentication. Can also be set via `SUPERMETAL_PASSWORD` environment variable.
- `skip_validation` (Boolean) Skip server-side validation of source and sink configurations before create/update. Can also be set via `SUPERMETAL_SKIP_VALIDATION` environment variable.
- `username` (String) Username for basic authentication. Can also be set via `SUPERMETAL_USERNAME` environment variable.
